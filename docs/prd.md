Documento de Requisitos do Produto (PRD) – NoCode Folio
Versão: 3.0 (Nomenclatura PT-BR + Int8)
1. Visão Geral (Overview)
O projeto consiste no desenvolvimento do NoCode Folio, uma aplicação web estilo "Link-in-Bio" baseada em grid modular (Bento Grid). O sistema permite que usuários criem páginas pessoais contendo "blocos" de conteúdo variado (Links, Textos, Imagens, Mapas).
O objetivo pedagógico é ensinar a construção de aplicações flexíveis utilizando colunas JSONB para dados polimórficos e estruturação de layout CSS Grid, tudo sob a identidade visual "Dark/Neon" da NoCode Startup.
2. Metas do Projeto
Grid Responsiva: O layout deve se adaptar automaticamente: 1 coluna (Mobile), 2 colunas (Tablet), 4 colunas (Desktop).
Flexibilidade de Dados: Utilizar JSONB para permitir que diferentes tipos de blocos (ex: um Link vs. um Mapa) compartilhem a mesma tabela no banco de dados.
Identidade Visual: Design "Glassmorphism" no modo escuro, utilizando Shadcn/UI e Tailwind CSS.
Performance: Carregamento otimizado utilizando React Query (TanStack Query) e chaves numéricas (int8) para indexação rápida.
3. Stack Tecnológica
Frontend: React, Vite, TypeScript.
UI Library: Shadcn/UI + Tailwind CSS + Lucide React (Ícones).
Gerenciamento de Estado: TanStack Query (React Query).
Backend (BaaS): Supabase.
Banco de Dados: PostgreSQL.
Autenticação: Supabase Auth.
Storage: Supabase Storage (Imagens).
4. Estrutura de Páginas e Fluxo
A. Perfil Público (Rota: /:slug)
Visão do Visitante: Acessa a URL (ex: folio.io/joao) e visualiza a grid de blocos.
Header: Foto do perfil, Nome, Bio e Redes Sociais.
Corpo: Renderização dos blocos ativos ordenados conforme definido pelo usuário.
Interação:
Blocos de Link: Abrem nova aba.
Blocos de Cópia (ex: Chave Pix/Email): Copiam para área de transferência com feedback visual.
B. Modo de Edição (Área Logada)
Acesso: O usuário logado vê um botão flutuante "Editar Grid" ao acessar seu próprio perfil.
Gerenciamento de Blocos:
Adicionar: Modal para escolher o tipo de bloco (Link, Imagem, Texto, Youtube, Mapa).
Editar Conteúdo: Formulário dinâmico baseado no tipo do bloco (atualiza o JSONB).
Excluir: Remove o bloco da grid.
Ordenação: Botões ou Drag-and-Drop para reordenar os itens visualmente.
Dados do Perfil: Formulário para editar foto, nome, bio e slug da URL.
C. Onboarding (Cadastro)
Fluxo de Login/Cadastro via Magic Link ou Google.
Trigger automático (ou fluxo no front) para criar a linha na tabela perfis assim que o usuário se cadastrar.

6. Segurança (RLS - Row Level Security)
As políticas de segurança devem garantir que apenas o dono do perfil possa editar seus dados.
Políticas da tabela perfis
Leitura (SELECT): true (Público - Qualquer um pode ver).
Criação (INSERT): auth.uid() = usuario_id (Usuário logado cria para si mesmo).
Edição (UPDATE): auth.uid() = usuario_id (Apenas o dono edita).
Políticas da tabela blocos
Como blocos não tem a coluna usuario_id direta, precisamos fazer um "join" na verificação.
Leitura (SELECT): true (Público).
Modificação (INSERT/UPDATE/DELETE):
SQL

exists (
  select 1 from perfis
  where perfis.id = blocos.perfil_id
  and perfis.usuario_id = auth.uid()
)


7. UI/UX Guidelines (Identidade NoCode Startup)
Tema Global: Dark Mode obrigatório (class="dark").
Background: bg-slate-950 (Fundo profundo).
Componente Bloco (Card):
Cor: bg-slate-900/40 (Semi-transparente).
Efeito: backdrop-blur-md (Vidro fosco).
Borda: border border-slate-800.
Arredondamento: rounded-3xl (Bordas muito redondas são a assinatura visual do estilo Bento).
Hover: hover:border-violet-500/50 (Brilho roxo sutil ao passar o mouse).
Botões de Ação: Gradiente from-violet-600 to-fuchsia-600.
Tipografia: Fonte Inter ou Plus Jakarta Sans.
8. Instruções para o Desenvolvedor (Cursor/Lovable)
Ao criar componentes, verifique sempre se o campo conteudo do JSONB possui as chaves necessárias antes de renderizar, para evitar erros de undefined.
Utilize a biblioteca clsx ou tailwind-merge para classes condicionais nos tamanhos dos blocos (col-span-1, col-span-2, etc).
Implemente Optimistic Updates com React Query ao reordenar os blocos, para que a interface não pisque enquanto salva no banco.
